# scene_16

## 固定役割

本書の残りの価値を伝え、自然に読書行動へつなげる。  
「もっと詳しく知りたい方は、本書を手に取ってみてください」という上品な案内をする。

scene_16は購入リンク誘導ではなく、本書を自然に案内するシーン。

## 固定してよい要素

- 本書の残りの価値
- 本を手に取る
- さらに深く学ぶ
- 読書の余韻
- 落ち着いた読書空間
- Book Baseらしい上品な案内

## 固定してはいけない要素

- 今回の本のタイトル
- 今回の著者名
- 今回の重要ポイントの具体語句
- Amazon
- 楽天
- 購入はこちら
- 概要欄リンク
- 今すぐ買う
- 売り込み感の強いCTA

## 可変データ構造

Codexはscene_16用に毎回以下を作成する。

```json
{
  "scene": 16,
  "fixed_role": "本書の残りの価値案内",
  "book_title": "現在の本のタイトル",
  "book_cover_path": "現在のbook_cover画像パス。使用しない場合はnull",
  "remaining_value_label": "本書に残っている価値を短く表す言葉",
  "read_invitation_label": "自然な読書案内を短く表す言葉",
  "visual_mode": "generic_book / real_cover_composite / reading_nook / open_book / desk_still_life のいずれか",
  "exact_text_elements": ["短い文字1", "短い文字2"]
}
```

## ブックカバーの扱い

### 実ブックカバーを使う場合

`input/YYYYMMDD_book_cover.*` が存在し、scene_16で使う場合は `visual_mode = real_cover_composite` にする。

- ブックカバーをAIに再生成させない。
- 表紙文字を崩さない。
- 実画像を後処理で合成する。
- scene_03より小さく、控えめに扱う。

### 実ブックカバーを使わない場合

- タイトルのない一般的な本、開いた本、しおり、ノート、読書灯、静かな机で表現する。
- 架空の表紙やタイトルを描かせない。
- 現在の本のタイトルをAIに描かせない。

## 画像内テキストルール

画像内テキストは2〜3個以内。基本は次のいずれか。

- 「本書を手に取る」「さらに深く学ぶ」
- 「本書の続きへ」「もっと深く知る」

長い固定文、概要欄リンク、購入表現、販売サイト名、英語テキスト、原稿本文全文は禁止。

## visual_mode の選び方

- `generic_book`：参照画像なし、表紙を崩したくない、読書の余韻を出したい場合。
- `real_cover_composite`：book_cover画像が存在し、表紙を控えめに見せたい場合。
- `reading_nook`：本を手に取りたくなる余韻を出し、売り込み感を避けたい場合。
- `open_book`：本の中にまだ学びが残っていることを表現したい場合。
- `desk_still_life`：本、ノート、ペン、しおりなどで上品な静物構図にする場合。

## 推奨構図

- 構図A：開いた本、しおり、ノート、読書灯。短いテキスト「さらに深く学ぶ」。
- 構図B：机の上の本に手がそっと伸びる。売り込みではなく自然な案内。
- 構図C：本の横に小さなカード。「本書の続きへ」「もっと深く知る」。
- 構図D：実カバーを右または左に控えめに合成。背景は落ち着いた読書机。
