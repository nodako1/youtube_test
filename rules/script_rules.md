# Book Base 原稿作成時チェックリスト

更新日：2026年6月17日

このチェックリストは、Book BaseのYouTube原稿を作成する前後に確認するための最新版ルールである。細かい表現ルールよりも、動画としての分かりやすさ、納得感、視聴維持、会社員目線での実用性を優先する。

## 1. 出力形式・文字数

- 原稿は必ず【シーン1】〜【シーン20】の20シーン構成にする。シーン数を増減させない。
- シーン見出しは必ず【シーン1】の形式にする。「シーン切り替え」「シーン切り替え：1」「## Scene 01」のような表記は使わない。
- 1シーンあたり180〜220字を目安にし、全体は3600〜4400字を目安にする。
- 文字数を満たすためだけの水増し文は入れない。短い場合は具体例、会社員の現実、本の重要論点、補強データを追加する。長い場合は重複、汎用的な感想、なくても伝わる説明を削る。
- 原稿本文を出力するときは、【シーン〇】だけ改行する。各シーンの本文は、シーン内で改行せず、1段落で詰める。
- 箇条書きや空行を入れず、そのまま読み上げられる形にする。
- タイトル、タイムスケジュール、35文字説明、コメントは、ユーザーから求められた時だけ原稿本文とは別に出力する。

## 2. 全体構成

- シーン1：統計、ニュース、調査データを使ったクイズ。選択肢はABC表記にし、最後は基本的に「それでは正解を発表します。」で締める。
- シーン2：クイズの正解発表から今回のテーマへ接続する。
- シーン3：本紹介と動画全体の結論。固定文「今回紹介するのは、〇〇さんの『〇〇』こちらの本になります。」を必ず入れる。
- シーン4：著者紹介と3つの重要ポイントの提示。
- シーン5〜8：重要ポイント①。シーン5の冒頭は必ず「重要ポイントの1つ目は〜」で始める。
- シーン7：重要ポイント①を補強する論文、研究、公的データ、科学的知見。
- シーン8：重要ポイント①の流れに絡めたチャンネル登録CTA。固定文を変えずに入れる。
- シーン9〜12：重要ポイント②。シーン9の冒頭は必ず「重要ポイントの2つ目は〜」で始める。
- シーン11：重要ポイント②を補強する有名人、経営者、歴史上の人物などの確認済み実話エピソード。
- シーン12：重要ポイント②の流れに絡めたコメント促進CTA。
- シーン13〜16：重要ポイント③。シーン13の冒頭は必ず「重要ポイントの3つ目は〜」で始める。
- シーン15：重要ポイント③を補強する確認済みの名言。
- シーン16：本書の残りの価値と本書案内。固定文「もし気になった方は、ぜひ本書を手に取ってみてください。」を入れ、概要欄リンクへの誘導は入れない。
- シーン17：この動画のおさらい。冒頭は必ず「今回の内容をおさらいすると〜」から始める。
- シーン18：仕事や日常への実践。
- シーン19：過去動画、関連動画への接続。なぜその動画とつながるのかを明確にし、「気になる方は、ぜひそちらもあわせてご視聴ください」のように次の行動を示す。
- シーン20：最終メッセージと締め。固定文を変えずに入れる。
- 重要ポイント①②③は独立した話ではなく、「問題提起 → 原因・背景 → 解決策・実践」の1本の流れにする。

## 3. 固定文・禁止表現

- シーン8の固定文：「このチャンネルでは、話題の本を、日々の仕事や年収アップにどう活かせるかという視点でお届けしています。興味のある方は、ぜひチャンネル登録お願いします。」
- シーン12のコメント促進は「みなさんも～～～した経験はありますか？もし似たような経験があれば、ぜひコメント欄で教えてください。」の形式にする。
- シーン16の固定文：「もし気になった方は、ぜひ本書を手に取ってみてください。」
- シーン20の固定文：「参考になった方は、高評価やハイプで応援していただけると嬉しいです。それでは、また次回の動画でお会いしましょう。最後までご視聴ありがとうございました。」
- シーン1では「選択肢は、」「1つ目」「2つ目」「3つ目」「少し考えてみてください。」を使わない。
- 短い感想だけの一文、無理な個性演出、下手なブラックジョーク、理解を助けない比喩、唐突な上司・納期・会議の例え話は入れない。

## 4. 事実確認・会社員目線

- 統計、ニュース、調査データは信頼できる出典を確認し、出典名だけでなく調査会社・団体がどんな組織かを簡潔に説明する。
- シーン7の研究・データ、シーン11の実話、シーン15の名言は、可能な限り一次情報または信頼できる出典で確認する。確認できない場合は別の確認可能な要素に差し替える。
- ユーザーが提供した要約やメモの重要論点を反映し、本が言っていないことを断定しない。
- 抽象語は会社員の日常に落とし込む。評価、年収、納期、会議、上司、部下、報連相、転職、副業、管理職などは、原稿の流れに自然につながる場合だけ使う。
- どの本にも使える汎用的なAI原稿にせず、その本ならではの概念、言葉、主張を残す。
- 読み上げたときに自然な一人語りになっているかを確認する。

## 5. 投稿補助情報

- タイムスケジュール、35文字説明、コメントはユーザーから求められた時だけ出力する。
- タイムスケジュールは0:00、2:00、4:00、6:00、8:00の5項目にする。
- 35文字説明は35文字以内を厳守し、必要に応じて文字数を併記する。
- コメントは3行構成にする。1行目は「今回は、〇〇について紹介しました。」、2行目は視聴者も経験していそうなことを質問し、3行目は「もし似たような経験があれば、ぜひコメント欄で教えてください。」にする。

## 6. Book Base文体・読み上げ品質ルール

原稿は単なる本の要約ではなく、会社員が「自分の仕事に関係ありそう」「続きが気になる」と感じるYouTubeナレーションとして書く。20シーン構成や文字数を守るだけでは不十分で、説明文・レポート調ではなく、自然な語りかけを優先する。

- 会社員が自分ごと化できる具体場面を入れる。会議、報連相、上司への相談、後輩への指摘、同僚との雑談、評価面談、納期前のやり取り、チャットでの一言、チーム内の空気などは、流れに自然につながる場合だけ使う。
- 一文の長さに変化をつける。短い文、中くらいの文、少し長い文を混ぜ、聞きやすいリズムを作る。
- 語尾を単調にしない。「です」「ます」だけを連発せず、「なんです」「ではないでしょうか」「かもしれません」「ということです」「見えてきます」「につながります」などを自然に混ぜる。
- 接続詞は機械的に「また」「さらに」「そして」を連発しない。「ここで大事なのは」「つまり」「たとえば」「一方で」「ここが怖いところで」「会社員の感覚で言うと」「ここから分かるのは」「だからこそ」「逆に言えば」「実は」「ここで本書が教えてくれるのは」などを、文脈に合わせて使う。
- 視聴者に語りかける。「こんな経験、ありませんか？」「少し想像してみてください」「仕事でもよくありますよね」「ここ、かなり大事です」「ここで勘違いしやすいのが」などを自然に使う。ただし、疑問文は1シーンに多くても1つまでにする。
- 「会社員の仕事に役立ちます」「ビジネスで活用できます」「人間関係が良くなります」のような汎用表現で終わらせず、小さな仕事場面に落とし込む。

### AIっぽい表現の禁止

以下のような表現は、機械的で弱いため避ける。出典が確認できないのに「研究で示されています」「検証済みです」と断定しない。

- 〜が効果的と言われています。
- 〜が特徴です。
- 〜について探っていきます。
- 〜ヒントが満載です。
- 〜第一人者です。
- 〜検証済みです。
- 〜とされています。
- 〜の重要性が示されています。
- 〜と言われています。
- 〜可能になるのです。

### scene_07 / scene_11 / scene_15 の根拠確認

- scene_07は研究・データで補強する。可能な範囲で具体的な研究、調査、公的データに寄せる。出典が弱い場合は断定せず `NEEDS_REVIEW` にする。
- scene_11は実話エピソードで補強する。確認可能な人物・出来事だけを使う。確認できない場合は、実在企業や人物名を出さず、別の検証可能な事例に差し替える。
- scene_15は名言で補強する。発言者、原文、文脈を確認する。確認できない場合は断定せず `NEEDS_REVIEW` にする。

### 原稿生成プロセス

一発生成で完成原稿を出さず、内部で次の工程に分ける。

1. Step 1：入力ファイルから、本のテーマ、著者、読者の悩み、中心メッセージ、会社員への接続点、3つの重要ポイントを抽出する。
2. Step 2：最新ルールに沿って、20シーンの役割を決める。
3. Step 3：各シーン180〜220字で初稿を作る。
4. Step 4：一文の長さ、語尾、句読点、接続詞、語りかけ、会社員目線、次シーンへの流れ、AIっぽさ除去の観点でBook Base文体にリライトする。
5. Step 5：20シーン、各180〜220字、全体3600〜4400字を検証する。
6. Step 6：文字数、形式、文体のどれかがNGなら再リライトする。

### 完成前の文体チェック

完成前に以下を5点満点で自己採点し、平均4点未満なら再生成する。

1. 読みたくなる言い回しか
2. 1文の長さに変化があるか
3. 句読点の位置が自然か
4. 接続詞が機械的ではないか
5. 語尾が単調ではないか
6. 会社員が自分ごと化できるか
7. AIっぽい言い回しが残っていないか
8. 本の内容に沿っているか
9. sceneごとの役割が明確か
10. 次のシーンを聞きたくなる流れがあるか
