# タイトル作成ルール

## 絶対形式

- タイトル案はA/B/Cの3案を出す。
- 各タイトルは必ず `【冒頭フック】メインタイトル【後方フック】` の形式にする。
- 前後どちらか片方だけの `【】` はNG。
- `【】` 内のフックは前後も含めて重複させない。

## 3パターンの役割

- Pattern A：脅し・損失回避型。放置リスク、見えない損失、逆効果になりそうな場面を具体的に見せる。
- Pattern B：誘惑・ベネフィット型。視聴後に得られる改善、評価、伝わりやすさ、仕事での使いやすさを具体的に見せる。
- Pattern C：逆張り・好奇心型。一般的な思い込みをひっくり返し、「え、そうなの？」と思わせる。ただし釣りにしない。

## 弱いフック禁止

以下のフック語は単独使用禁止。使う場合も必ず具体化する。

- 知らないと損
- 保存版
- 誰でもできる
- 会社員必見
- 仕事がラクに
- 意外な秘訣
- 知っておかないと損
- 驚きの効果
- すぐ使える
- 人間関係が変わる
- 今すぐ見て
- 完全解説
- まとめ

## 具体性と数字

- 抽象語だけで終わらせない。「信頼を得る方法」「驚きの効果」のような汎用表現で締めない。
- 本の個性が出る言葉を入れる。言い換え事典なら、NG表現、改善表現、角が立つ言葉、印象を悪くする言い方、信頼を失う話し方などを優先する。
- 数字化できる場合は、5選、7つ、10例、20連発、NGワード7選などを優先する。
- 誇大表現や本の内容からズレる断定は避ける。

## 生成前手順

タイトル作成前に必ず以下を内部で行う。

1. 本の核心を一言で整理する。
2. 会社員が困る場面を整理する。
3. この本ならではの具体ポイントを洗い出す。
4. 数字化できる要素を探す。
5. 損失回避 / ベネフィット / 逆張りの3方向に分ける。
6. 前後の `【】` フックを別々に作る。
7. 最低3回候補を作り、最もルールに合うものだけを出力する。
8. 出力直前に弱いフック語が入っていないか自己チェックする。
