# Book Base 動画内画像 ブロック構造ルール

## 参照順序

画像プロンプトを作るときは、必ず次の順で参照する。

1. `script_rules.md`
2. `image_rules_common.md`
3. `image_block_rules.md`
4. 該当する `image_scene_rules/scene_XX.md`
5. 現在のシーン原稿
6. 前後シーンの画像メタ情報

## ブロック1：シーン1〜4 導入・結論・全体像

- 役割：視聴者に問題意識を持たせ、動画を見る理由を作り、今回の本から何が学べるかを短時間で示す。
- 流れ：シーン1は問い・問題提起、シーン2は結論・本の紹介、シーン3は全体像の提示、シーン4は重要ポイントへの橋渡し。
- 画像方針：入口として分かりやすく、視聴者の悩みとつながる。シーン1では答えを出さず、シーン2で答えと本の価値を示す。
- 避けること：冒頭から情報過多、シーン1で結論を出す、シーン2で本の紹介感が弱い、シーン3〜4が地味すぎる。

## ブロック2：シーン5〜8 重要ポイント1

- 役割：動画本編の最初の柱。視聴者がまず理解すべき前提・土台・最初の気づきを扱う。
- 流れ：シーン5は導入、シーン6は説明・理由、シーン7は具体化・日常への接続、シーン8は締め＋登録促し。
- 画像方針：土台感、入口感、最初の一歩、基礎、気づき、日常の違和感を出す。
- 向いている表現：仕事机、ノート、タスク、時計、スマホ、会社員の悩みや気づき。
- 避けること：重要ポイント2・3と同じ温度感、いきなり応用や総まとめ、派手な登録ボタン画像。

## ブロック3：シーン9〜12 重要ポイント2

- 役割：重要ポイント1で作った前提をもとに、理由・構造・比較・具体例を通じて理解を一段深める。
- 流れ：シーン9は導入、シーン10は理由・構造、シーン11は具体例・比較、シーン12は締め＋コメント促し。
- 画像方針：一段深く理解する、構造が見える、理由が分かる、なるほどと腹落ちする。
- 向いている表現：比較、構造、原因と結果、Before/After、2択、悪循環と好循環、ノート整理、会議や仕事場面。
- 避けること：重要ポイント1と同じ入口感、重要ポイント3のような最終実践感、文字だらけの図解、コメント欄UIの再現。

## ブロック4：シーン13〜16 重要ポイント3

- 役割：重要ポイント1・2を踏まえ、視聴者が自分はどうすればいいかを考えられるようにする。
- 流れ：シーン13は導入、シーン14は実践・応用、シーン15は整理・行動化、シーン16は締め。
- 画像方針：実践、応用、行動、内省、着地。重要ポイント1・2より落ち着かせ、生活や仕事へ置き換えられるようにする。
- 向いている表現：小さな一歩、行動開始、整理されたデスク、チェックリスト、予定表、朝の光、静かな内省。
- 避けること：入口感、説明・比較だけ、文字だらけの整理画像、CTAや外部リンク誘導の雰囲気、最終まとめとの混同。

## ブロック5：シーン17〜20 おさらい・実践・関連動画接続・締め

- 役割：重要ポイント1・2・3を踏まえて本全体の意味を回収し、視聴者に余韻を残す。
- 流れ：シーン17は動画内容のおさらい、シーン18は学びの実践化・余韻、シーン19は関連動画接続、シーン20はまとめ・本書の残りの価値案内。
- 画像方針：本編全体を回収し、3つの要点が積み上がった結果として見える。シーン19は自然な関連動画接続、シーン20はすっきりした着地にする。
- 向いている表現：俯瞰、本を閉じる、学びがつながる、道や橋、整ったノート、朝の光、静かな前向きさ、積み上がった3つの要点。
- 避けること：ただの説明画像、重要ポイント3と同じ実践画像、露骨な広告、情報過多。

## 品質チェック項目

- シーン番号に合った役割で画像化されているか。
- 所属ブロックを正しく理解しているか。
- シーン5〜8が重要ポイント1、シーン9〜12が重要ポイント2、シーン13〜16が重要ポイント3として表現されているか。
- 重要ポイント1・2・3に理解の流れがあり、単なる並列になっていないか。
- 重要ポイント1は土台・入口感、重要ポイント2は理解深化・構造理解、重要ポイント3は実践・応用・行動化になっているか。
- シーン17〜20で本編全体が回収されているか。
- 前後の画像と構図が被っていないか。
- 原稿本文をそのまま長文で載せていないか。
- 1画像1メッセージになっているか。
- Book Baseらしい水彩画風・高級感があるか。
