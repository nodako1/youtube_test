# Book Base 投稿補助情報 出力ルール

## 目的

Book Baseの動画ごとに、YouTube投稿時に必要となるタイトル、タイムスケジュール、50文字説明、コメントを生成する。これらは原稿生成後に作成し、原稿本文には混ぜず、`metadata.md`として別ファイル出力する。

## metadata.md 出力順

1. タイトル案
2. タイムスケジュール
3. 50文字説明
4. コメント

## タイトル作成ルール

- ABテスト用に3パターン作成する。
- Pattern A：脅し・損失回避型。
- Pattern B：誘惑・ベネフィット型。
- Pattern C：逆張り・好奇心型。
- タイトルは必ず前後に`【】`を入れ、`【冒頭フック】メインタイトル【後方フック】`形式にする。
- `【】`内のフック文言は毎回オリジナルにし、過去タイトルや過去フックを使い回さない。
- 弱いフック（【知らないと損】【保存版】【誰でもできる】【会社員必見】【今すぐ見て】【完全解説】【まとめ】）は単独で使わない。
- 数字を入れられる動画では、5選、3つの習慣、10選、月3万円、1日5分などの数字を検討する。ただし内容とズレる場合は使わない。

## タイムスケジュール作成ルール

- 必ず5項目だけで作成する。
- 時間は必ず0:00、2:00、4:00、6:00、8:00で固定する。
- 出力形式は以下にする。
  - `0:00 本日のテーマ：〇〇〇〇`
  - `2:00 ポイント①：〇〇〇〇`
  - `4:00 ポイント②：〇〇〇〇`
  - `6:00 ポイント③：〇〇〇〇`
  - `8:00 まとめ：〇〇〇〇`
- 「シーン1〜4」などの内部表記は出さない。

## 50文字説明 作成ルール

- 必ず50〜60文字に収める。
- 出力時は必ず文字数を併記する。
- 本のテーマ、動画で得られる学び、会社員が自分ごと化できる価値を含める。
- 1文で自然に読める説明にする。
- 抽象的すぎる説明や煽りすぎる説明は避ける。

## コメント作成ルール

- コメントは必ず3行構成にする。
- 1行目：`今回は、〇〇について紹介しました。`
- 2行目：視聴者も経験していそうな質問にする。
- 3行目：`もし似たような経験があれば、ぜひコメント欄で教えてください。`
- キーワード型コメント促進にしない。

## quality_report.md に記録する内容

投稿補助情報チェックとして、タイトルA/B/C、タイトル前後の【】、過去タイトル重複、過去フック重複、タイトルの誇張、タイムスケジュール5項目、時間固定、内部シーン表記なし、50文字説明50〜60文字、文字数併記、コメント3行構成、コメント3行目固定文、コメント内容と動画テーマの一致を確認する。
